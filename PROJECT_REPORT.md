# AI回复质量自动评估与错误归因分析项目报告

## 一、项目概述

本项目围绕 AI 训练师日常工作中的“模型回复质量评估”场景展开，目标是搭建一套可批量处理模型回答样本的自动评估流程。

项目基于 Python 实现，主要功能包括：

1. 读取用户问题与模型回答样本；
2. 从用户指令中动态解析显性约束规则；
3. 对模型回答进行规则检测；
4. 按多个维度生成评分；
5. 自动判断回答是否合格；
6. 自动归因错误类型；
7. 输出 Excel 多 Sheet 分析报表；
8. 生成 Markdown 项目评估报告。

本项目重点模拟 AI 训练师在真实工作中的质检流程，即不只是判断单条回答是否合格，还要进一步分析模型回答中高频出现的问题，并形成可反馈给研发、算法或产品团队的优化建议。

---

## 二、项目背景

在大模型应用场景中，模型回答质量直接影响用户体验。AI 训练师或模型质检人员通常需要对模型输出进行评估，包括判断回答是否遵守用户指令、是否完整、是否格式正确、是否自然、是否存在事实错误等。

传统人工评估虽然准确性较高，但存在以下问题：

* 重复性工作较多；
* 明确规则类问题人工判断效率低；
* 评估结果不易批量统计；
* 错误类型需要人工整理；
* 难以快速发现模型高频问题。

因此，本项目尝试将部分明确规则类任务自动化处理，例如：

* 用户要求写 3 条，模型是否真的写了 3 条；
* 用户要求每条不超过 15 字，模型是否超字数；
* 用户要求必须包含某个关键词，模型是否遗漏；
* 用户要求不要出现某个词，模型是否违反限制；
* 用户要求不要解释，模型是否添加了多余说明；
* 用户要求用表格，模型是否按照表格形式输出。

对于自然度、事实准确性、语义相关性等难以完全依靠规则判断的内容，本项目保留人工复核和后续接入大模型辅助评估的扩展空间。

---

## 三、项目目标

本项目的主要目标包括：

1. 搭建一个可运行的 AI 回复质量评估脚本；
2. 将人工评估标准转化为可执行的规则检测逻辑；
3. 实现从用户输入中动态解析规则，而不是将测试样本写死；
4. 对模型回答进行多维度评分；
5. 自动输出合格 / 不合格结果与错误类型；
6. 生成可视化 Excel 分析报表；
7. 输出项目评估报告，为模型优化提供参考。

## 五、评估指标设计

本项目采用五个维度对模型回答进行评分。

| 评分维度  | 说明                         |
| ----- | -------------------------- |
| 指令遵循  | 判断模型是否遵守用户的关键词、禁用词、禁止解释等要求 |
| 回答完整性 | 判断模型是否完整完成用户要求             |
| 格式规范  | 判断回答条数、字数、表格结构等是否符合要求      |
| 表达自然度 | 判断回答是否存在明显夸张、生硬或过度复杂的问题    |
| 准确性   | 判断明确规则下的结论是否正确             |

每个维度采用 1 到 5 分评分方式：

| 分数  | 含义      |
| --- | ------- |
| 5 分 | 完全符合要求  |
| 4 分 | 基本符合要求  |
| 3 分 | 部分符合要求  |
| 2 分 | 存在明显问题  |
| 1 分 | 严重不符合要求 |

系统会根据五项评分计算综合得分，并结合核心维度进行合格判断。

## 六、核心功能实现

### 1. 动态规则解析

项目最重要的优化点是从“写死规则”升级为“动态规则解析”。

第一版脚本中，部分规则采用固定判断方式，例如：

```python
if "必须包含卧室" in user_input:
    ...
```

这种方式只能处理特定样本，泛化能力较弱。

优化后，脚本通过正则表达式和关键词规则，从用户指令中动态提取约束条件，例如：

```text
用户要求：写3条建议，每条不超过15字，不要出现新手
```

系统可解析为：

```python
{
    "item_count": 3,
    "max_len_per_item": 15,
    "forbidden_words": ["新手"]
}
```

目前可解析的规则包括：

* 条数要求：写 3 条、列出 5 点、生成 2 个；
* 字数限制：每条不超过 15 字、30 字以内；
* 必须包含关键词：必须包含卧室、必须包含感应灯和磁吸；
* 禁用词限制：不要出现新手、不能包含刺眼；
* 禁止解释：不要解释、只给答案；
* 表格要求：用表格列出、表格形式；
* 简单表达要求：简单解释、通俗解释；
* 语气要求：不要夸张、避免夸张表达。

---

### 2. 规则检测

解析出规则后，系统会对模型回答进行检测，包括：

* 检查条数是否符合要求；
* 检查每条字数或整体字数是否超限；
* 检查是否包含必需关键词；
* 检查是否出现禁用词；
* 检查是否存在多余解释；
* 检查是否符合表格格式；
* 检查是否存在夸张表达；
* 检查明确判断题是否判断正确。

---

### 3. 多维度评分

脚本会根据各项检测结果生成五个自动评分：

```text
auto_instruction_score
auto_completeness_score
auto_format_score
auto_naturalness_score
auto_accuracy_score
```

然后计算综合得分：

```text
auto_total_score
```

合格判断逻辑综合考虑：

* 综合得分是否达到标准；
* 指令遵循是否达标；
* 格式规范是否达标。

这样可以避免某些回答虽然整体分数较高，但违反核心用户指令仍被判为合格的问题。

---

### 4. 错误类型归因

项目会根据失败原因自动归因错误类型。

主要错误类型包括：

| 错误类型      | 说明                |
| --------- | ----------------- |
| 指令违背      | 缺少关键词、出现禁用词、多余解释等 |
| 格式 / 长度错误 | 条数、字数、表格格式不符合要求   |
| 回答不完整     | 没有完成用户全部要求        |
| 表达不符合要求   | 语气夸张、表达复杂或不自然     |
| 判断错误      | 对明确规则类问题判断错误      |
| 其他错误      | 暂未归入具体类型的问题       |
| 无明显错误     | 未发现明显问题           |

---

### 5. Excel 分析报表

项目会生成 Excel 多 Sheet 分析报表，文件路径为：

```text
output/auto_eval_result.xlsx
```

Excel 报表包含以下 Sheet：

| Sheet 名称 | 内容                        |

| 错误类型统计   | 各类错误数量和占比                 |
| 评分维度统计   | 五项评分维度的平均分、最低分、最高分        |
| 低分样本     | 自动筛选不合格或低分样本，便于人工复核       |
| 明细数据     | 每条样本的完整评估结果               |

图表分析中包含：

* 合格 / 不合格数量对比图；
* 合格率分布饼图；
* 错误类型分布图；
* 五项评分维度平均分图；
* 低分样本错误类型分布图。

## 八、结果分析

### 1. 合格率分析

本次测试样本合格率为 70.00%，说明大部分模型回答能够满足用户基本要求。

但仍有 30.00% 的样本被判定为不合格，说明在限制类任务中，模型仍容易出现没有严格遵守用户要求的问题。

常见问题包括：

* 用户要求不要出现某个词，但模型仍出现；
* 用户要求必须包含关键词，但模型遗漏；
* 用户要求控制字数，但模型超出限制；
* 用户要求列出固定条数，但模型输出条数不足；
* 用户要求简单解释，但模型回答过于复杂。

---

### 2. 指令遵循问题分析

指令遵循是 AI 回复质量评估中的核心指标。

本项目中，指令违背类错误主要包括：

* 缺少必须包含的关键词；
* 出现用户明确禁止的词；
* 在“不要解释”的任务中添加额外说明；
* 未遵守用户对语气的限制。

这类问题在 AI 训练师工作中优先级较高，因为用户已经明确给出限制条件，模型未遵守即属于明显不合格。

---

### 3. 格式与长度问题分析

格式与长度问题主要集中在：

* 每条字数超过限制；
* 总字数超过限制；
* 条数不符合要求；
* 用户要求表格，但回答没有表格结构。

这类问题适合通过规则自动评估，因为判断标准明确，人工复核成本较低。

项目中使用 Python 规则检测可以有效减少人工对这类明确约束任务的重复判断。

---

### 4. 回答完整性问题分析

回答不完整主要表现为：

* 用户要求列出多个点，但模型只给出概括性回答；
* 用户要求同时包含定义和作用，但模型只回答了定义；
* 用户要求总结 3 点原因，但模型只给出笼统描述。

这类问题反映模型在多要求任务中的拆解能力不足。

---

### 5. 表达自然度问题分析

表达自然度较难完全通过规则判断。

本项目目前主要检测明显问题，例如：

* 用户要求简单解释，但模型使用过多专业词；
* 用户要求不要夸张，但模型出现“秒杀”“必约”“超强”等表达。

更细致的自然度判断仍需要人工复核或大模型辅助评估。

---

### 6. 准确性问题分析

准确性是最难完全规则化的指标之一。

本项目目前仅处理部分明确判断题，例如：

```text
用户要求写3条，模型只写2条，判断是否合格
```

对于这类规则明确的问题，系统可以自动判断模型结论是否正确。

但对于开放问答、事实判断和复杂语义理解问题，仍需要人工复核或接入大模型评估。

---

## 九、项目价值

本项目的价值主要体现在以下几个方面：

### 1. 提升明确规则类任务评估效率

对于条数、字数、关键词、禁用词、表格等明确规则，Python 自动检测比人工逐条判断更高效、更稳定。

### 2. 支持批量错误归因

项目不仅判断是否合格，还能自动输出错误类型和判断理由，方便后续统计分析。

### 3. 便于形成模型优化反馈

通过错误类型统计和低分样本分析，可以发现模型高频问题，并反馈给研发或算法团队。

### 4. 适合 AI 训练师工作流

项目流程贴近 AI 训练师真实工作内容：

```text
样本整理
→ 回复评估
→ 错误归因
→ 数据统计
→ 问题总结
→ 优化建议
```

### 5. 具备后续扩展空间

当前项目已经实现规则评估基础能力，后续可以继续扩展为线上质检工具或接入大模型评估框架。

## 十、项目局限

本项目目前仍存在以下局限：

1. 对自然度、事实准确性、语义相关性的判断能力有限；
2. 规则解析依赖正则和关键词，无法覆盖所有自然语言表达；
3. 复杂多轮对话场景尚未支持；
4. 当前规则词表仍写在代码中，尚未配置化；
5. 代码初版功能集中在单个文件中，后续需要模块化拆分；
6. 缺少单元测试和异常处理；
7. 暂未接入大模型评估或人工复核系统。
