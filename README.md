# AI回复质量自动评估与错误归因分析项目

## 一、项目简介

本项目是一个面向 AI 训练师 / AI 回复质检场景的自动评估工具，主要用于批量评估大模型回复质量。

项目基于 Python 实现，从用户指令中动态解析显性约束规则，例如条数要求、字数限制、必须包含关键词、禁用词、是否需要表格、是否禁止解释等，再对模型回答进行自动检测、多维度评分、错误类型归因，并输出 Excel 分析报表和 Markdown 评估报告。

本项目适用于以下场景：

* AI 回复质量评估
* 指令遵循能力检测
* 模型回答错误归因
* AI 训练师标注结果辅助分析
* Prompt 优化效果评估
* 模型高频问题统计分析

---

## 二、项目背景

在 AI 训练师和模型质检工作中，常见任务不仅是判断模型回答是否合格，还需要进一步分析模型为什么出错、错误集中在哪些类型、哪些维度表现较弱，并将分析结果反馈给研发、算法或产品团队，用于后续模型优化。

传统人工评估虽然准确，但存在效率低、重复劳动多、统计分析不便等问题。因此，本项目尝试将部分明确规则类任务自动化处理，例如：

* 用户要求“写3条”，模型是否真的写了3条
* 用户要求“每条不超过15字”，模型是否超字数
* 用户要求“必须包含卧室”，模型是否漏掉关键词
* 用户要求“不要出现新手”，模型是否出现禁用词
* 用户要求“不要解释”，模型是否添加了多余解释
* 用户要求“用表格列出”，模型是否符合表格结构

对于自然度、事实准确性、语义相关性等难以完全规则化判断的部分，项目保留人工复核或后续接入大模型评估的扩展空间。

## 三、项目功能

### 1. 动态规则解析

项目不是将测试样本规则写死，而是通过正则和关键词解析，从用户指令中动态提取评估规则。

可解析规则包括：

* 条数要求：如写3条、列出5点、生成2个
* 字数限制：如每条不超过15字、30字以内
* 必须包含关键词：如必须包含卧室、必须包含感应灯和磁吸
* 禁用词限制：如不要出现新手、不能包含刺眼
* 禁止解释：如不要解释、只给答案
* 表格要求：如用表格列出、表格形式
* 简单易懂要求：如简单解释、通俗解释
* 语气要求：如不要夸张、避免夸张表达

---

### 2. 多维度评分

项目对每条模型回答进行五个维度评分：

| 评分维度               | 含义                           |
| ------------------ | ---------------------------- |
| instruction_score  | 指令遵循能力，判断是否遵守关键词、禁用词、禁止解释等限制 |
| completeness_score | 回答完整性，判断是否完整满足用户要求           |
| format_score       | 格式规范，判断条数、字数、表格结构是否符合要求      |
| naturalness_score  | 表达自然度，判断是否存在明显夸张、复杂、生硬表达     |
| accuracy_score     | 准确性，判断明确规则下的回答是否正确           |

系统会进一步计算综合得分，并根据评分结果判断回答是否合格。

---

### 3. 错误类型归因

项目会根据检测结果自动归因错误类型，包括：

* 指令违背
* 格式 / 长度错误
* 回答不完整
* 表达不符合要求
* 判断错误
* 其他错误
* 无明显错误

错误归因结果可用于分析模型高频问题，辅助后续 Prompt 优化、训练数据补充和人工复核。

---

### 4. Excel 多 Sheet 分析报表

项目会自动生成 Excel 分析文件，包含多个 Sheet：

| Sheet 名称 | 内容                         |
| -------- | -------------------------- |
| 汇总       | 总样本数、合格数量、不合格数量、合格率、各维度平均分 |
| 图表分析     | 合格率分布、错误类型分布、评分维度分布等图表     |
| 错误类型统计   | 各类错误数量和占比                  |
| 评分维度统计   | 五项评分维度的平均分、最低分、最高分         |
| 低分样本     | 自动筛选不合格或低分样本，便于人工复核        |
| 明细数据     | 每条样本的完整评估结果                |

Excel 中包含柱状图、饼图、条件格式和基础样式美化，方便查看整体评估情况。

---

### 5. Markdown 评估报告

项目会自动生成 Markdown 格式评估报告，内容包括：

* 项目概况
* 整体评估结果
* 五项指标平均分
* 错误类型统计
* 典型错误样本
* 优化建议

该报告可用于项目复盘、面试展示或提交给研发 / 产品团队作为模型优化参考。

## 八、核心流程

项目整体流程如下：

```text
读取CSV样本
        ↓

        ↓
```

## 九、核心设计思路

本项目的核心设计不是简单判断模型回答是否合格，而是将 AI 训练师的人工评估流程结构化、规则化和数据化。

第一版脚本曾采用固定规则判断，例如直接判断“是否包含卧室”“是否出现新手”。后续发现这种方式泛化能力较弱，因此改为动态规则解析方式。

优化后的脚本可以从用户指令中自动提取变量，例如：

```text
用户要求：写3条建议，每条不超过15字，不要出现新手
```

系统解析为：

```python
{
    "item_count": 3,
    "max_len_per_item": 15,
    "forbidden_words": ["新手"]
}
```

再使用解析结果对模型回答进行检测。

这种方式比写死规则更适合真实业务场景，也方便后续扩展更多规则类型。

## 十、项目亮点

1. 支持从用户指令中动态解析规则，而不是针对固定样本写死判断逻辑。
2. 支持多维度评分，包括指令遵循、完整性、格式、自然度和准确性。
3. 支持错误类型自动归因，便于统计模型高频问题。
4. 自动生成 Excel 多 Sheet 分析报表，包含汇总、图表、错误统计、评分维度和低分样本。
5. 自动生成 Markdown 评估报告，可用于项目展示或模型优化复盘。
6. 项目逻辑贴近 AI 训练师真实工作流程，强调质检、归因、统计和优化反馈。

## 十一、当前局限

本项目主要适合评估明确规则类任务，例如条数、字数、关键词、禁用词、格式要求等。

对于以下问题，纯规则评估仍存在局限：

* 回答是否真正有帮助
* 语气是否足够自然
* 内容是否存在隐性事实错误
* 回答是否充分理解用户意图
* 开放式回答的语义相关性判断

因此，本项目更适合作为自动初筛工具，而不是完全替代人工评估。

## 十二、后续优化方向

后续可以从以下方向继续优化：

1. 规则配置化
   将禁用词、复杂词、解释性词语、夸张词等迁移到配置文件，减少代码修改成本。

2. 模块化重构
   将规则解析、检查函数、评分归因、Excel 报表、Markdown 报告拆分成独立模块，提升代码可维护性。

3. 增加日志系统
   使用 logging 记录文件读取、样本处理、异常样本和输出路径。

4. 增加异常处理
   针对 CSV 字段缺失、空回答、文件路径错误、Excel 文件占用等情况增加处理逻辑。

5. 增加测试用例
   为条数、字数、关键词、禁用词、表格要求等规则增加单元测试，避免后续修改影响已有功能。

6. 接入大模型评估
   对自然度、事实准确性、语义相关性等规则难以覆盖的部分，可接入 DeepEval、Promptfoo 或大模型裁判进行辅助评估。
